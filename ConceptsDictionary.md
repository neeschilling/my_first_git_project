# Dictionary of Concepts

## Differentiating

- Git: software to create repositories that track changes

- GitHub: Hub where all the repositories are stored for all the different projects

## Conceptual areas

1. Developing area: My folder where my project is developed.
2. Staging area: organize the snapshot, useful with multiple files 
3. git local repository: keeps your version, also create points in time (timeline)

## Git commit editor

1. Vi
2. Nano
3. VSCode

## Collaborations

- How to start: Go to setting of the project inGitHub, add the persson. Once they accept the collaboration email your are set.

- Why branchs for collaborations: To work on different solutions. To minimize overlaps and conflicts when working in the same file.

p.s.: add your initials to the branch name. EX.: BP_branch_name
